What is Dependency Injection?

Dependency Injection (DI) is a design pattern where objects receive their dependencies from external sources rather than creating them internally. This promotes loose coupling, easier testing, and better code organization.

❌ Without DI

Tightly Coupled Code
Hard to Test
class_name Player
extends Node2D

var health_system: HealthSystem
var inventory: Inventory

func _ready():
    # Creating dependencies internally
    health_system = HealthSystem.new()
    inventory = Inventory.new()
    
func take_damage(amount: int):
    health_system.reduce_health(amount)

✅ With DI

Loosely Coupled Code
Easy to Test
class_name Player
extends Node2D

var health_system: HealthSystem
var inventory: Inventory

# Dependencies injected via constructor
func _init(health: HealthSystem, inv: Inventory):
    health_system = health
    inventory = inv
    
func take_damage(amount: int):
    health_system.reduce_health(amount)

Implementation Methods

1. Constructor Injection

Constructor Injection Example
Preferred Method
class_name WeaponSystem
extends Node

var damage_calculator: DamageCalculator
var sound_manager: SoundManager

func _init(calc: DamageCalculator, sound: SoundManager):
    damage_calculator = calc
    sound_manager = sound

func fire_weapon(target: Node):
    var damage = damage_calculator.calculate(base_damage)
    sound_manager.play_sound("weapon_fire")
    target.take_damage(damage)

2. Setter Injection

Setter Injection Example
Optional Dependencies
class_name GameUI
extends Control

var analytics: Analytics

func set_analytics(analytics_service: Analytics):
    analytics = analytics_service

func on_button_clicked():
    if analytics:
        analytics.track_event("button_click")

🏗️
Simple DI Container

DIContainer.gd
Service Locator
class_name DIContainer
extends RefCounted

static var _instance: DIContainer
var _services: Dictionary = {}

static func instance() -> DIContainer:
    if not _instance:
        _instance = DIContainer.new()
    return _instance

func register(service_name: String, service):
    _services[service_name] = service

func get_service(service_name: String):
    if service_name in _services:
        return _services[service_name]
    else:
        push_error("Service not found: " + service_name)
        return null

Best Practices

  • ✅ Prefer constructor injection for required dependencies
  • ✅ Use interfaces/abstracts for better decoupling
  • ✅ Keep your constructors simple and focused
  • ✅ Don't inject too many dependencies (consider facade pattern)
  • ✅ Use DI containers for complex dependency graphs
  • ✅ Document your dependencies clearly
